The Core Pattern: Closed-Loop Orchestration

"You do not prompt your sub-agents. You prompt your primary agent, and the primary agent prompts your sub-agents. They respond back to the primary agent, which synthesizes the information for you."

The General Contractor Model

  • • Orchestrator holds the "master blueprint"
  • • Sub-Agents are specialized trades
  • • Sub-Agents report to contractor, not user
  • • Direct user communication forbidden

Minimum Viable Stack

  • • Claude Opus 4.5 - Orchestrator + Sub-Agents
  • • Claude Code (CLI) - Agent harness
  • • Task Tool - Native sub-agent spawning
  • • Markdown File - Master plan

The Five Architectural Pillars

Click each pillar to explore its principle, practice, and payoff:

Pillar I: Communication

"Agents communicate through closed loops and verifiable artifacts."

The Principle

Sub-agents report exclusively to the Orchestrator. Direct user communication is forbidden.

The Practice

  • • Condense: Summaries, metrics
  • • Preserve: Files, logs, errors

The Payoff

Verifiable workflows. Task completion confirmed by artifact presence.

Pillar II: Execution

"Execution is a choice: parallel for speed, sequential for logic."

The Principle

Orchestration adapts to task dependencies. Independent = parallel, dependent = sequential.

The Practice

  • • Parallel: 5 agents testing 5 stories
  • • Sequential: Plan → Build → Test

The Payoff

Hybrid workflows: sequential master plan with parallel bursts.

Pillar III: Empowerment & Isolation

"Empower agents with tools, but isolate them in sandboxes."

The Principle

Provide "right tooling" for capability. Restrict environment to contain risk.

The Practice

  • • Comprehensive tool suites
  • • E2B sandboxes per agent
  • • "Private Workshop" model

The Payoff

Agents do "nasty engineering work" with contained blast radius.

Pillar IV: State & Handoffs

"State lives in the environment, not the agent's memory."

The Principle

Preserve work artifacts and environment, not conversational history.

Four Artifacts

  • • Environment (Sandbox)
  • • Instructional (Master Plan)
  • • Result (Logs, Assets)
  • • Codebase (Brownfield)

The Payoff

Perfect recall. Seamless multi-session handoffs.

Pillar V: Resilience

"Resilience is architected through isolation and error routing."

The Principle

Single sub-agent failure must not derail orchestration. Contain blast radius.

The Practice

  • • Isolated sandboxes
  • • Error routing loops
  • • Self-correcting workflows

The Payoff

Self-healing system. Component failures are expected, not exceptional.

The Meta-Goal

"The goal is not to build the application. It is to build the system that builds the application."

We are no longer prompt engineers talking to a single model. We are system architects designing resilient, scalable, observable agentic frameworks. The agent is the new compositional unit of engineering.
